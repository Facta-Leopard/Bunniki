Bunniki 개인정보처리방침

최종 업데이트: 2026년 4월 30일

Bunniki는 사용자가 직접 선택한 로컬 문서와 폴더를 읽어 위키를 생성하고, 탐색하고, 질의할 수 있게 돕는 macOS 및 iPadOS 앱입니다. 또한 챗봇 모드를 켜면 로컬 위키를 검색하지 않고 선택한 Provider에 일반 채팅 요청을 보낼 수 있습니다. Bunniki는 사용자가 선택하지 않은 위치를 임의로 탐색하지 않으며, 원본 소스 파일을 수정하지 않습니다.

앱이 처리하는 데이터

  • 사용자가 선택한 소스 파일과 폴더의 텍스트 및 파일 메타데이터
  • 생성된 위키 문서, 실행 기록, 인덱스, 캐시, 진단 정보
  • 사용자가 입력한 질의, 위키 기반 답변에서 사용되는 위키 문맥, 대화 기록, 생성 지시
  • 선택한 Provider 설정, API 키, OAuth 토큰, 선택형 Provider 계정 식별자
  • iPadOS에서 광고가 표시될 때 Google Mobile Ads SDK가 처리할 수 있는 광고 관련 데이터

로컬 저장

Bunniki는 사용자가 선택한 출력 라이브러리와 앱 지원 폴더에 데이터를 저장합니다. Provider 인증 정보는 Apple CryptoKit 기반 AES-GCM으로 암호화된 로컬 저장소에 보관됩니다. 이 저장소는 Apple Keychain은 아니며, 사용자의 기기 안에서만 앱이 사용합니다.

원격 Provider 사용

OpenAI API, Codex OAuth, 사용자가 설정한 OpenAI 호환 네트워크 endpoint를 사용할 경우, Bunniki는 기능 제공을 위해 질의, 대화 기록, 위키 문맥, 생성 지시, 사용자가 선택한 문서의 발췌를 해당 Provider로 전송할 수 있습니다. 챗봇 모드에서는 로컬 위키 검색을 건너뛰고 위키 발췌 대신 사용자의 메시지와 최근 대화 기록을 포함한 일반 채팅 요청을 전송합니다. Bunniki는 네트워크 기반 Provider를 사용하기 전에 앱 안에서 이를 고지하고 사용자의 동의를 받습니다.

Apple Foundation Models Provider는 사용 가능한 경우 온디바이스 모델을 사용합니다.

광고 및 추적

Bunniki의 iPadOS 버전은 Google Mobile Ads SDK를 사용해 광고를 표시할 수 있습니다. Google Mobile Ads SDK는 운영체제에서 허용되는 범위의 기기 식별자, IP 주소에서 추정되는 위치 또는 대략적인 위치 정보, 광고 상호작용, 진단 및 성능 데이터 등 광고 관련 데이터를 광고 제공, 측정, 부정 사용 방지, 규정 준수, 그리고 허용되는 경우 광고 목적을 위해 처리할 수 있습니다.

Bunniki는 사용자의 소스 문서, 생성된 위키 페이지, 질의 내용, Provider 인증 정보, 로컬 작업공간 콘텐츠를 광고 목적으로 사용하지 않으며, 사용자가 선택한 문서 내용을 광고 타기팅 목적으로 광고 SDK에 제공하지 않습니다. macOS 버전은 Google Mobile Ads 대신 자체 후원 및 제휴 문의 배너를 표시할 수 있습니다.

데이터 삭제

사용자는 앱 안에서 질의 기록, 실행 기록, 위키, 소스 연결, Provider 인증 정보, 앱 상태를 삭제할 수 있습니다. 출력 라이브러리에 저장된 파일은 Finder에서도 직접 삭제할 수 있습니다.
