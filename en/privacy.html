Bunniki Privacy Policy

Last updated: April 24, 2026

Bunniki is a macOS app that helps users generate and query a wiki from local documents and folders selected by the user. Bunniki does not scan locations the user has not selected and does not modify original source files.

Data the App Processes

  • Text and file metadata from source files and folders selected by the user
  • Generated wiki documents, run history, indexes, caches, and diagnostics
  • User questions, wiki context, and generation instructions
  • Selected Provider settings, API keys, OAuth tokens, and optional Provider account identifiers

Local Storage

Bunniki stores data in the output library selected by the user and in the app support folder. Provider credentials are stored in a local encrypted store using Apple CryptoKit AES-GCM. This store is not Apple Keychain, and it is used locally on the user's Mac.

Remote Providers

When OpenAI API, Codex OAuth, or a user-configured OpenAI-compatible network endpoint is used, Bunniki may send the user's question, wiki context, generation instructions, and excerpts from user-selected documents to that Provider to provide the requested feature. Bunniki discloses this in the app and asks for consent before using network-based Providers for document or query processing.

Apple Foundation Models Provider uses on-device models when available.

Tracking and Advertising

Bunniki does not include advertising tracking, third-party advertising SDKs, or remote analytics SDKs. The in-app banner is a first-party sponsorship and partnership inquiry area.

Data Deletion

Users can delete query history, run history, wikis, source links, Provider credentials, and app state in the app. Files in the selected output library can also be deleted directly in Finder.

Contact

For privacy questions, contact shteosis@gmail.com.
