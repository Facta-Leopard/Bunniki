Bunniki Privacy Policy

Last updated: April 30, 2026

Bunniki is a macOS and iPadOS app that helps users generate, browse, and query a wiki from local documents and folders selected by the user. It can also answer in Chatbot Mode, which sends a regular chat request to the selected Provider without searching the local wiki. Bunniki does not scan locations the user has not selected and does not modify original source files.

Data the App Processes

  • Text and file metadata from source files and folders selected by the user
  • Generated wiki documents, run history, indexes, caches, and diagnostics
  • User questions, wiki context when wiki-based answers are used, conversation history, and generation instructions
  • Selected Provider settings, API keys, OAuth tokens, and optional Provider account identifiers
  • Ad-related data processed by Google Mobile Ads SDK on iPadOS when ads are shown, as described below

Local Storage

Bunniki stores data in the output library selected by the user and in the app support folder. Provider credentials are stored in a local encrypted store using Apple CryptoKit AES-GCM. This store is not Apple Keychain, and it is used locally on the user's device.

Remote Providers

When OpenAI API, Codex OAuth, or a user-configured OpenAI-compatible network endpoint is used, Bunniki may send the user's question, conversation history, wiki context, generation instructions, and excerpts from user-selected documents to that Provider to provide the requested feature. In Chatbot Mode, Bunniki skips local wiki search and sends a normal chat request with the user's message and recent conversation history instead of wiki excerpts. Bunniki discloses remote Provider use in the app and asks for consent before using network-based Providers for document or query processing.

Apple Foundation Models Provider uses on-device models when available.

Advertising and Tracking

The iPadOS version of Bunniki may display ads using Google Mobile Ads SDK. Google Mobile Ads SDK may process ad-related data such as device identifiers where permitted by the operating system, IP-derived or coarse location information, ad interactions, and diagnostics or performance data for ad delivery, measurement, fraud prevention, compliance, and, where permitted, advertising purposes.

Bunniki does not use source documents, generated wiki pages, user questions, Provider credentials, or local workspace content for advertising, and Bunniki does not share user-selected document contents with the advertising SDK for ad targeting. The macOS version may show a first-party sponsorship and partnership inquiry banner instead of Google Mobile Ads.

Data Deletion

Users can delete query history, run history, wikis, source links, Provider credentials, and app state in the app. Files in the selected output library can also be deleted directly in Finder.
